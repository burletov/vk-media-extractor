# MediaTidy: продуктовый и технический план

Дата оценки: 15 июня 2026 года.

## Название

Рабочее название: **MediaTidy**.

Оно описывает основную ценность продукта — привести локальную медиатеку в порядок — и не ограничивает приложение одной социальной сетью. Импорт из VK лучше оставить названием функции: «Импорт из VK».

Запасные варианты:

- FrameNest — более брендовый, но хуже объясняет очистку.
- MediaSweep — хорошо объясняет поиск мусора, но звучит агрессивнее.
- ClearFrame — подходит для фото, но слабее охватывает видео и документы.

Перед публичным запуском нужно отдельно проверить товарные знаки, домены и магазины приложений в целевых странах. Обычный веб-поиск не является юридической проверкой доступности названия.

## Позиционирование

MediaTidy — локальный менеджер фото и видео:

- ищет точные и визуально похожие файлы;
- помогает сравнить экземпляры и безопасно убрать лишнее в карантин;
- классифицирует медиа только как подсказку;
- импортирует вложения из внешних источников;
- не отправляет личную медиатеку в облако.

Главное отличие от обычного duplicate finder: единый рабочий процесс «импорт → анализ → сравнение → безопасная сортировка».

## Рынок

Существующие продукты подтверждают спрос, но закрывают разные части сценария:

- [Duplicate Cleaner Pro](https://www.digitalvolcano.co.uk/dcpro.html) силён в хешах, фильтрах, профилях, папках и файловых операциях.
- [Gemini 2](https://macpaw.com/gemini) продаёт простой интерфейс, Smart Select, мониторинг новых дубликатов и безопасное восстановление; официальная цена начинается с $19.95.
- [PhotoSweeper](https://overmacs.com/?p=photosweeper) предлагает серии снимков, видео, авто-выбор, EXIF, сравнение и пакетное переименование.
- [Czkawka](https://github.com/qarmin/czkawka) показывает, что у бесплатного кроссплатформенного инструмента может быть крупная open-source аудитория.

Вывод: конкурировать только «поиском дубликатов» поздно. Нужны сильный UX, приватность, импорт источников и прозрачные рекомендации.

## Монетизация

Рекомендуемая модель на первом этапе:

1. Бесплатное локальное ядро с открытым исходным кодом.
2. Добровольная поддержка: GitHub Sponsors и обычные платёжные способы; криптовалюта — дополнительный, а не единственный канал.
3. Позже — разовая лицензия Pro, ориентир $19–29, без обязательной подписки.

Бесплатно:

- локальное сканирование;
- точные и похожие файлы;
- карантин и восстановление;
- базовая модель;
- ручная сортировка.

Pro:

- дополнительные импортёры;
- усиленные и специализированные модели;
- наблюдение за папками;
- автоматические профили и расписания;
- расширенная обработка RAW/видео;
- коммерческое использование и приоритетная поддержка.

Подписка оправдана только после появления постоянных серверных расходов или регулярно обновляемого облачного сервиса. Для локальной утилиты, которую запускают периодически, обязательная подписка создаст лишнее сопротивление.

## Лицензия и распространение

Для настоящего open source репозиторию нужна явная лицензия; без неё действует обычное авторское право. Это прямо указано в [документации GitHub](https://docs.github.com/en/repositories/managing-your-repositorys-settings-and-features/customizing-your-repository/licensing-a-repository).

Практичный выбор:

- **MPL-2.0** для открытого ядра: изменения открытых файлов должны оставаться открытыми, но коммерческие модули можно держать отдельно.
- отдельная коммерческая лицензия для Pro-компонентов и корпоративного использования.

Структура GitHub:

- `MediaTidy.Core` — сканирование, хеши, модели, операции с файлами;
- `MediaTidy.Desktop` — интерфейс;
- `MediaTidy.Platform.*` — OCR, превью и системная интеграция;
- `docs`, `THIRD_PARTY_NOTICES`, политика приватности и модель угроз;
- GitHub Actions для тестов, сборок и подписанных релизов.

Бинарные файлы можно размещать в GitHub Releases: [GitHub разрешает файлы до 2 GiB и не устанавливает общий лимит размера релиза или трафика](https://docs.github.com/en/repositories/releasing-projects-on-github/about-releases). Модели лучше выпускать отдельными опциональными пакетами.

[GitHub Sponsors](https://docs.github.com/en/sponsors/getting-started-with-github-sponsors/about-github-sponsors) поддерживает разовые и ежемесячные платежи; доступность получения средств зависит от региона.

## Распознавание

В приложение добавлена переключаемая CLIP ViT-B/16 INT8. ONNX-модель имеет размер около 87.5 МБ и доступна из [репозитория Xenova](https://huggingface.co/Xenova/clip-vit-base-patch16/tree/main/onnx).

Важно: более крупная модель не гарантирует корректность именно наших семи классов. [Модельная карточка CLIP](https://huggingface.co/openai/clip-vit-base-patch16) прямо требует тестирования на фиксированной предметной области. До автоматических действий нужны:

1. размеченный набор хотя бы из 500–1000 пользовательских файлов;
2. матрица ошибок по каждому классу;
3. отдельные пороги для документов, чеков, мемов и размытия;
4. возможность исправить категорию и использовать исправления для калибровки;
5. сохранение «Не определено» при слабом или противоречивом результате.

Следующий кандидат после измерений — SigLIP2 или специализированный классификатор, дообученный на собственном наборе. Сначала нужен benchmark, потом увеличение размера модели.

## Linux и macOS

Текущий WinForms-код нельзя напрямую выпустить для Linux/macOS: он использует Windows Forms, Windows OCR и Windows Shell API.

Рекомендуемый путь — Avalonia. [Официальная документация Avalonia](https://docs.avaloniaui.net/docs/supported-platforms) указывает поддержку Windows, macOS и desktop Linux; macOS-сборки можно кросс-компилировать, но подпись и notarization всё равно требуют Apple-инфраструктуры.

Этапы:

1. Выделить `MediaTidy.Core` под `net8.0`: поиск файлов, хеши, группировка, ONNX, карантин, импорт VK.
2. Ввести интерфейсы `IOcrService`, `IThumbnailService`, `IFileNavigator`, `IVideoMetadataService`.
3. Windows: Windows OCR и Shell thumbnails.
4. Linux/macOS: Tesseract или PaddleOCR, FFmpeg/FFprobe и SkiaSharp.
5. Перенести UI на Avalonia с общими ViewModel.
6. Пакеты: Windows MSIX/EXE, macOS DMG с подписью/notarization, Linux AppImage + deb.
7. CI на Windows, Ubuntu и macOS; тесты x64 и ARM64.

Реалистичная оценка для одного разработчика: 4–8 недель до первого кроссплатформенного beta-релиза, затем отдельная стабилизация упаковки, OCR и системной интеграции.

## Следующие полезные функции

Приоритет 1:

- сравнение двух файлов рядом с синхронным увеличением;
- «лучший кадр» в серии по резкости, лицам и закрытым глазам;
- сохранённые профили сканирования;
- возобновляемый импорт и очередь неудачных загрузок;
- экспорт отчёта до любых файловых операций.

Приоритет 2:

- импорт Telegram export, Google Takeout и локальных архивов;
- поиск RAW/JPEG пар и Live Photo;
- географические и временные группы;
- ручное исправление категории и локальная калибровка;
- наблюдение за папками без автоматического удаления.

Приоритет 3:

- плагины источников;
- правила пакетного переименования;
- дедупликация через hard links для опытных пользователей;
- семейная библиотека и перенос между дисками с проверкой хеша.

## Метрики перед продажей

- доля неверных рекомендаций «оставить»;
- precision/recall по каждой категории;
- число восстановлений из карантина;
- время сканирования на 10 000 файлов;
- процент успешных и возобновлённых импортов VK;
- конверсия бесплатной версии в разовую Pro-лицензию;
- количество активных пользователей через 30 дней без сбора личных путей и содержимого файлов.
